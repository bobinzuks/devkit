- README: Sponsors first, features second

**starship (53K stars):**
- Description: "minimal, blazing-fast, infinitely customizable"

## Keyword Density Recommendations

### Primary Focus (Use Frequently):
- Rust (3-5x in description/README)
- CLI / command-line (5-8x)
- developer tools (2-3x)
- offline (2-3x)
- fast/blazing (1-2x - overuse is cliché)

### Secondary Focus (Use Naturally):
- privacy, security (1-2x each)
- cross-platform (1-2x)
- Features: base64, uuid, hash, json (1x each in intro)

### Avoid Keyword Stuffing:
- Don't repeat exact phrases
- Use variations: "command-line" vs "terminal" vs "CLI"
- Natural language > SEO optimization

## GitHub SEO Best Practices

1. **Repository Name**: Short, memorable, keyword-rich
   - Current: "devkit" ✓ Perfect

2. **Description** (350 char limit): Front-load keywords
   - First 3 words most important
   - Include "Rust" and "CLI" early

3. **README First Paragraph**: Critical for SEO
   - Repeat top keywords
   - Include all features

4. **Topics**: Use all 20 slots
   - Mix broad + specific
   - Update quarterly based on trends

5. **Star Growth**: Signals to GitHub algorithm
   - Encourage stars via good README
   - "Star if useful" CTAs

## Recommendations Summary

**Primary Strategy**: Position as "The Rust alternative to online dev tools"

**Secondary Angle**: "Privacy-focused, blazing-fast, single-binary developer utilities"

**Differentiation**: Offline + Security + Speed + Rust reliability

**Target Audience**:
- Security-conscious developers
- Remote workers
- DevOps engineers
- Privacy advocates
- Rust enthusiasts
- Terminal power users

**Avoid**:
- Generic "utility tool" positioning
- Over-emphasizing features vs benefits
- Competing on feature count
- Ignoring the Rust ecosystem brand

**Emphasize**:
- Speed (Rust's strength)
- Security (offline execution)
- Simplicity (single binary)
- Privacy (no data leaving machine)
- Convenience (no compilation for buyers)
